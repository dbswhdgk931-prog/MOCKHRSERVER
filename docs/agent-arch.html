5. Power Automate Action 설계

Flow A · 공통 데이터 수집

입력: EmployeeId

동작:

  • Dataverse 6개 테이블 병렬 조회 (Employee, Education, Career, OverseasExp, Family, Certification)
  • Employee 기준 FK 매칭으로 관련 레코드 수집
  • 결과를 Slide 1 매핑용 JSON으로 조합

출력: CommonDataJSON (Slide 1 전체 데이터)

Flow B · 평가/리더십 데이터 수집

입력: EmployeeId, Years (기본 3개년)

동작:

  • Evaluation 테이블: 3개년 업적/역량 등급 조회
  • EvalComment 테이블: 3개년 평가 코멘트 원본 조회
  • LeadershipSurvey 테이블: 동료/부하별 점수 + 코멘트

출력: EvalDataJSON (Slide 2/3 정형 부분)

Flow C · AI 역량 분석 (핵심)

입력: EvalDataJSON, ProfileType, CompetencyCriteria

동작:

  • Dataverse에서 ProfileType에 맞는 CompetencyCriteria 조회
  • ProfileType별 AI Builder 프롬프트 분기:
    • 그룹/DY → ELP 9개 역량 점수 채점 + 코멘트 + Short 판정
    • 케미칼 → 역량전문성/준비도/종합의견 서술
  • 역량 기준(가중치)을 프롬프트에 포함하여 일관된 채점

출력: AIAnalysisJSON (Slide 2 or 3 역량모델 부분)

Flow D · 문서 생성

입력: CommonDataJSON, EvalDataJSON, AIAnalysisJSON, ProfileType, OutputFormat

동작:

  • SharePoint에서 PPT 템플릿 로드 (ProfileType별 다른 Slide 2/3)
  • Slide 1: 공통 데이터 매핑 (텍스트 치환)
  • Slide 2/3: ProfileType에 따라 그룹DY용 or 케미칼용 매핑
  • 완성 파일 SharePoint 저장 → 공유 링크 반환

출력: DocumentUrl (SharePoint 링크)

6. Dataverse 테이블 분리 구조 (핵심)

설계 원칙: 결과양식 PPT의 각 "칸"이 어떤 테이블의 어떤 컬럼에서 오는지 1:1 추적이 가능하도록 설계합니다. 인사 시스템(inHR/myHR) 연동을 가정하여, 실제 API 응답 구조와 유사하게 정규화합니다.

[과제]
위 데이터를 기반으로 각 역량 항목에 대해 아래 형식으로 채점하세요.

[출력 형식 — JSON]
{
  "shortDecision": "Recommend" 또는 "Not Recommend",
  "shortReason": "판정 근거 1~2문장",
  "totalScore": 가중합산 점수,
  "competencies": [
    {
      "domain": "CEO 핵심 역량",
      "name": "Value",

    ...9개 역량 모두

}

- 코멘트는 반드시 3개년 데이터의 구체적 내용을 인용
- 가중합산 점수 = Σ(역량별 점수 × 가중치)
